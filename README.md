## 💥 트러블 슈팅

### 📌 header.html

 1. 버거 메뉴 클릭 후 스크롤시 하단에 여백이 생기는 이슈 발생

    *헤더가 아래로 스크롤시 숨겨지는 방식인데, 버거 메뉴 클릭한 상태에서도 적용되는 상황
    
    ⇒ **해결방법**: 버거 버튼에 active 클래스 여부를 확인하여, true일 경우 스크롤을 숨기게 하여 버거 메뉴가 보일 시 스크롤이 불가하게 작업함

### 📌 index.html

 1. 메인 슬라이드 텍스트 애니메이션이 한 번만 적용되는 이슈
  
    *첫 페이지 로드시 메인 슬라이드의 텍스트 애니메이션이 한 번만 적용되고, 슬라이드가 넘어가도 애니메이션이 다시 실행되지 않는 상황 발생
       
    ⇒ **해결방법**: swiper.on을 사용하여 슬라이드가 변경될 때마다 텍스트 애니메이션 효과가 다시 적용되도록 처리<br>

 2. RoomList 섹션에서 이미지 클릭 시 부모 요소의 너비가 비정상적으로 설정되는 문제
   
    *각 룸 이미지를 클릭했을 때, 부모 요소가 이미지의 너비만 반영해 콘텐츠가 정상적으로 노출되지 않는 상황
    
    ⇒ **해결방법**: active 클래스가 적용될 때 flex-shrink를 비활성화하고, 전체 너비를 설정하여 영역이 올바르게 확장되도록 수정<br>

 3. Special 섹션의 스크롤 이벤트가 반응형에서도 작동하는 이슈
       
    *반응형 뷰포트에서도 스크립트에 의한 스크롤 이벤트가 작동하여 레이아웃에 문제가 발생
    
    ⇒ **해결방법**: window.matchMedia를 사용해 조건에 맞는 화면 크기에서만 스크립트가 실행되도록 처리<br>

 4. 예약하기 바의 fixed가 적용되지 않는 문제

    *body에 translateY 값을 적용해 스크롤을 구현한 탓에, 예약하기 바의 fixed가 정상적으로 작동하지 않는 상황
    
    ⇒ **해결방법**: translateY 값을 body가 아닌 .wrap 클래스에 적용하고, 예약하기 바 관련 태그를 .wrap 바깥으로 이동시켜 fixed가 정상 작동하도록 수정<br>

 5. 메인 페이지에 Swiper 슬라이드가 두 번 이상 사용될 때 충돌하는 이슈

    *동일한 클래스명을 가진 Swiper 슬라이드를 페이지 내 두 번 사용하면서 충돌 발생
    
    ⇒ **해결방법**: 두 번째 슬라이드에는 다른 클래스명을 부여하여 Swiper 인스턴스가 독립적으로 작동하도록 처리
